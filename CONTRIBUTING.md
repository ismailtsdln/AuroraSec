# Contributing to AuroraSec

Thank you for your interest in contributing to AuroraSec! We appreciate your help in making this tool better for everyone.

## Getting Started

1. **Fork the Repository**: Create a fork of the AuroraSec repository on GitHub.
2. **Clone the Fork**: Clone your fork to your local machine.
3. **Install Dependencies**: Run `go mod download` to install the required Go modules.
4. **Create a Branch**: Create a new branch for your feature or bug fix.

## Development Guidelines

- **Go Standards**: Follow standard Go coding practices and formatting (`go fmt`).
- **Testing**: Ensure all new features or bug fixes include appropriate unit tests.
- **Modular Design**: When adding new security checks, create a new module in `internal/pkg/modules/`.

## Pull Request Process

1. **Self-Review**: Review your changes for clarity, performance, and security.
2. **Submit PR**: Open a pull request against the `main` branch of the original repository.
3. **Reference Issues**: Link your PR to any relevant issues.
4. **Feedback**: Be prepared to address feedback and make adjustments during the review process.

## Code of Conduct

Please be respectful and professional in all interactions within the AuroraSec community.
